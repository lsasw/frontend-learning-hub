🔷

Pyramid

介于 Django 与 Flask 之间的"中庸之道"，按需组装。"Start small, finish big" 理念。适合需要灵活但又不想要过于极简的团队。

中大型应用渐进增强社区较小
⚡

Litestar（原 Starlite）

FastAPI 的强力挑战者——原生 async、类视图、DI、内置 DTO 验证、性能极佳。API 设计与 FastAPI 类似但社区仍在快速成长中。

高性能 API类型完备生态年轻
🚀

Sanic

全球首个 Python 异步 Web 框架，类 Flask API 风格。"async everywhere" 的早期探索者，内置 WebSocket 支持，适合对异步体验有极致追求的项目。

实时服务异步原生生态不如 FastAPI
🌐

Falcon

专为 REST API 设计的极简轻量框架，无模板引擎、无 ORM，极致性能。适合构建纯粹的后端 API 网关或超高性能微服务。

纯 API 网关最低开销功能少
🧩

Bottle

单文件微框架（整个框架就一个 .py 文件），零依赖，极度轻量。适合嵌入式场景、简单 API 或教学用途。不适合中大型项目。

嵌入式/原型零依赖功能极简
🏗

Tornado

Facebook 开源的非阻塞异步框架，曾是 Python 异步编程的先锋。内置 HTTP 客户端 + WebSocket + 长轮询，适合需要长连接保持的场景。

长连接/WebSocket成熟稳定不再前沿
💡
快速选型建议：全栈 CMS/后台系统选 Django，微服务/高性能 API 选 FastAPI，简单原型/小型服务选 Flask。2024 年后新项目建议优先考虑 async-first 框架（FastAPI / Litestar）。
2 数据库与 ORM — 数据持久化

📦

PonyORM

用 Python 生成器表达式写查询，语法优雅到令人惊叹。select(p for p in Person if p.age > 18) 直接用 Python 表达式转 SQL。社区较小，生产慎用。

语法极优雅社区小、性能一般
🛠

SQLModel

FastAPI 作者 tiangolo 出品，融合 SQLAlchemy + Pydantic 的「二合一」模型——一个模型类同时完成数据库映射和数据验证。适合 FastAPI + SQLAlchemy 全栈。

FastAPI 官方推荐不够成熟
⚡

Piccolo

新兴异步 ORM，自带 Admin 后台面板和迁移工具，主打"比 Django ORM 更灵活、比 Tortoise 更完整"的中间路线。

内置 Admin生态极新
Python 
# SQLAlchemy 2.0 异步查询示例
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy import select

engine = create_async_engine("postgresql+asyncpg://...")

async def get_users_over(age: int):
    async with AsyncSession(engine) as session:
        stmt = select(User).where(User.age > age)
        result = await session.execute(stmt)
        return result.scalars().all()
3 异步框架与服务器 — 运行时载体

Python 服务器分两大阵营：传统的 WSGI（同步）和现代的 ASGI（异步）。选型取决于你的框架——同步框架用 WSGI 服务器，异步框架必须用 ASGI 服务器。

🦄

Uvicorn ASGI

基于 uvloop + httptools 的闪电级 ASGI 服务器，FastAPI 官方首选。单进程即可处理万级并发。生产通常配合 Gunicorn 做多进程管理。

极致性能FastAPI/Litestar 首选
🦖

Gunicorn WSGI

Ruby Unicorn 的 Python 版本，最成熟的 WSGI 服务器。"Green Unicorn"。核心价值是 process management —— 配合 Uvicorn worker 可同时管理异步工作进程。

进程管理王者纯 WSGI，需配合 worker
📡

Hypercorn ASGI

支持 HTTP/1、HTTP/2、HTTP/3（QUIC）、WebSocket，协议支持最全的 ASGI 服务器。可同时运行同步和异步应用。

协议支持最全性能略低于 Uvicorn
🏭

uWSGI WSGI

意大利炮级别的全能型应用服务器——支持多协议、多语言、多 worker 模式。配置复杂但功能极其强大，适合需要精细控制的生产环境。

功能最全面配置复杂、重
🎭

Daphne ASGI

Django Channels 官方 ASGI 服务器，专为 Django 的 WebSocket / HTTP/2 支持设计。如果你在用 Django Channels，这就是官方标配。

Django Channels 标配通用性不如 Uvicorn
🪟

Waitress WSGI

纯 Python 实现的 WSGI 服务器，跨平台无需 C 扩展。Windows 下生产部署的首选（Gunicorn 不支持 Windows），适合所有需要跨平台兼容的场景。

跨平台/Windows 首选性能一般
🔧
生产环境最佳实践：Gunicorn（进程管理）+ Uvicorn worker（每进程内异步）= 兼顾并发与管理。命令：gunicorn -w 4 -k uvicorn.workers.UvicornWorker app:app

5 缓存方案 — 提速利器

缓存是性能优化的第一站——从本地内存缓存到分布式 Redis，不同场景对应不同方案。

🗄 主流缓存方案一览

方案类型特点适用场景替代性
Redis分布式内存数据结构丰富（String/Hash/List/Set/SortedSet）、持久化、集群模式、发布订阅几乎所有场景 ★标配，很难被替代
Memcached分布式内存极致简单、纯 KV、多线程、内存管理优秀简单 KV 缓存，Redis 出现后逐渐边缘化Redis 的旧日对手
cachetools进程内内存纯 Python，支持 TTL/LRU/LFU/TTL 策略，零依赖单进程小数据缓存、无外部依赖场景内存缓存最简方案
diskcache磁盘缓存基于 SQLite 的磁盘持久化缓存，支持 LRU 淘汰大容量缓存、持久化本地缓存不想搭 Redis 时的大缓存方案
Python lru_cache函数级内存标准库 functools.lru_cache / cache，装饰器即用耗时计算的函数结果缓存零成本、最简单
Django Cache Framework框架级抽象统一 API，后端可切换 Redis/Memcached/DB/FileDjango 项目首选Django 生态标准
🔥

Redis 客户端: redis-py

Python 操作 Redis 的事实标准库，支持连接池、Pipeline、Pub/Sub、Lua 脚本。async 版本用 redis.asyncio，与 FastAPI 天然搭配。

生态最全标准选择
⚡

Redis 客户端: walrus

基于 redis-py 的轻量封装，提供 Cache/Lock/RateLimiter 等常用模式的高级抽象，减少样板代码。

高级抽象少写代码不如 redis-py 灵活
Python 
# Redis 缓存模式三种写法
import redis.asyncio as aioredis

redis = aioredis.Redis(from_url="redis://localhost")

# ① 简单的 SET + GET + TTL
await redis.set("user:1", user_json, ex=300)  # 5分钟过期
user = await redis.get("user:1")

# ② 缓存穿透防护：SETNX
if await redis.setnx("lock:task:1", 1):
    await redis.expire("lock:task:1", 10)
    # 执行业务...

# ③ 缓存雪崩：随机过期时间
import random
ttl = 300 + random.randint(0, 60)  # 5~6 分钟随机
await redis.setex("hot:data", ttl, value)
6 任务队列 — 异步解耦

耗时操作（发邮件、图像处理、报表生成）必须从请求线程中剥离。任务队列就是做这件事的——把"现在做不了的事情"放到后台慢慢做。

🌿

Celery — 元老级

Python 异步任务队列的代名词。背靠丰富的文档和社区生态，配置灵活、支持数十种 broker/backend 组合。缺点是配置复杂、错误排查费劲。

生态最全、最流行配置繁琐、调试难
⚡

Dramatiq — 轻量高效

Celery 的现代化替代品——API 简洁、自动重试、支持中间件、性能优秀。npm 的创建者开发。"比 Celery 好用 3 倍"。

API 简洁、性能好生态不如 Celery
🔄

Taskiq — 异步原生

专为 Python async/await 设计的任务队列。FastAPI 集成优秀，类型安全、依赖注入。async-first 项目（FastAPI/Litestar）的最新技术前沿。

async 原生、类型安全较新、文档少
Python 
# Celery — 经典写法
from celery import Celery

app = Celery('tasks', broker='redis://localhost:6379/0')

@app.task(bind=True, max_retries=3, default_retry_delay=60)
def send_welcome_email(self, user_id):
    try:
        user = get_user(user_id)
        email_service.send(user.email, "Welcome!")
    except Exception as e:
        # 失败后自动重试最多 3 次，每次间隔递增
        raise self.retry(exc=e)

# ── Taskiq — async 原生风格 ──
from taskiq import TaskiqScheduler

@broker.task
async def generate_report(user_id: int) -> str:
    data = await fetch_user_data(user_id)
    return await render_pdf(data)

7 认证与权限 — 安全防线

认证（你是谁）和授权（你能做什么）是后端安全的两根支柱。方案从简单的 JWT 到完整的企业级 OAuth2 / OIDC 体系，复杂度差异巨大。

🔐 认证方案全景对比

方案类型复杂度适用场景代表实现
JWT无状态令牌低微服务间认证、前后端分离、API 鉴权PyJWT / python-jose
OAuth2 + OIDC授权框架 + 身份层中高第三方登录、SSO 单点登录、企业级应用Authlib / OAuthLib
Session 认证有状态服务端低传统 MVC 网站、Django 管理后台Django Auth / Flask-Login
API Key静态令牌最低内部服务调用、第三方 API、机器间通信自定义中间件
Keycloak独立认证服务中需要完整 IDP 的企业级系统python-keycloak
Auth0 / ClerkSaaS 认证平台低不想自己写认证逻辑的团队对应 SDK
🎫

PyJWT

Python 最流行的 JWT 库，支持 HS256/RS256/ES256 多种算法，encode/decode 简单直接。FastAPI 中的 JWT 验证几乎都基于此库。

简洁、标准兼容JWT 首选
🔑

Authlib

Python OAuth/OIDC 最完整的实现，支持 OAuth 1.0/2.0、OIDC、JWT、JWS/JWE/JWK。提供 Flask/Django/Starlette 的集成方案。

OAuth 完整生态需要 OAuth2 时首选
🛡

FastAPI Users

FastAPI 的用户管理插件——注册、登录、密码重置、邮箱验证、OAuth 一键集成。省去手动写用户系统的麻烦。

FastAPI 开箱即用仅限 FastAPI
🔒

passlib

密码哈希的事实标准库，支持 bcrypt/argon2/scrypt/pbkdf2。不处理认证流程，只专注密码加密这一件事。

密码哈希专家所有项目通用
🏰

python-keycloak

Keycloak 的 Python 客户端，支持管理 API + OpenID Connect。如果企业在用 Keycloak 做统一认证，这就是 Python 端的标准对接方式。

企业级 Keycloak 集成依赖 Keycloak 服务
📋

Casbin / Oso

权限模型引擎——Casbin 支持 ACL/RBAC/ABAC 等所有常见模型，Oso 用声明式语言写权限规则。从认证到授权的下一步。

权限模型灵活复杂权限场景
Python 
# FastAPI + JWT 最小实现
from fastapi import FastAPI, Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer
import jwt

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="login")

def create_token(user_id: str):
    return jwt.encode(
        {"sub": user_id, "exp": datetime.utcnow() + timedelta(hours=24)},
        SECRET_KEY, algorithm="HS256"
    )

async def get_current_user(token = Depends(oauth2_scheme)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
        return payload["sub"]
    except jwt.ExpiredSignatureError:
        raise HTTPException(401, "Token expired")

📌 核心选型原则

  • 不要过度设计：小项目用 Django 太重、用 Flask 正好；反过来，大项目用 Flask 会陷入"拼拼图地狱"
  • 2024+ 新项目优先 async：FastAPI / Litestar > Flask > Django，除非你需要 Django Admin 或已有 Django 团队
  • ORM 选 SQLAlchemy 几乎不会错：它覆盖了 80% 的场景，是 Python 生态最强大、最成熟的 ORM，没有之一
  • Redis 是"默认缓存"：除非资源极其有限（此时可用本地 diskcache/cachetools），否则 Redis 的经验曲线和运维成本远低于它的价值
  • Celery 不是唯一：在新项目中，Taskiq（async）和 Dramatiq（简洁）都是比 Celery 更好的选择
  • 认证别自己造轮子：JWT + PyJWT 做 API 认证，Authlib 做 OAuth2/OIDC，FastAPI Users 做全功能用户管理，企业内用 Keycloak
🏢 企业级 Django 全栈

框架：Django 4.2+ / ORM：Django ORM / 服务器：Gunicorn + Uvicorn / 队列：Celery / 缓存：Redis / 认证：Django Auth + DRF + JWT
适合：ERP、CMS、SaaS 后台系统、有经典关系型数据模型的项目。

⚡ 高性能微服务 FastAPI

框架：FastAPI / ORM：SQLAlchemy 2.0 async / 服务器：Uvicorn + Gunicorn / 队列：Taskiq / 缓存：Redis / 认证：PyJWT / docs: 自动 OpenAPI
适合：纯 API 服务、微服务网关、实时数据接口、AI 模型服务。
