2 Spring Framework — 核心基础

Spring Framework 是整个生态的基石，提供了最核心的 IoC 容器和 AOP 机制。理解 Spring Framework 的原理，是深入 Boot 和 Cloud 的前提。

🔧 IoC 容器（控制反转 / 依赖注入）

核心思想：对象不再自己创建依赖，而是由 Spring 容器统一管理和注入。ApplicationContext 是容器的入口，通过 @Autowired / @Component / @Bean 等注解驱动。

@RestController

    public OrderController(OrderService orderService) {

✅
使用建议：优先使用构造器注入，避免字段注入（@Autowired 在字段上），因为构造器注入有利于单元测试和不可变性。Spring 4.3+ 单构造器时 @Autowired 可省略。

🎯 AOP（面向切面编程）

核心思想：将横切关注点（日志、事务、权限）从业务代码中抽离出来，通过"切面"织入目标方法，实现业务与基础设施的解耦。

切面 (Aspect)
@Aspect
织入→
切入点 (Pointcut)
匹配方法
触发→
通知 (Advice)
Before/After/Around
java 
@Aspect
@Component
public class LoggingAspect {

    // 拦截 com.example.service 包下所有方法
    @Around("execution(* com.example.service.*.*(..))")
    public Object logExecution(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = joinPoint.proceed();  // 执行目标方法
        long elapsed = System.currentTimeMillis() - start;
        log.info("{} took {}ms", joinPoint.getSignature(), elapsed);
        return result;
    }
}
💡
AOP 的典型应用：事务管理（@Transactional 底层就是 AOP）、日志记录、权限校验、性能监控、缓存处理。Spring AOP 默认使用 JDK 动态代理（接口）或 CGLIB（类）。

🗄️ 事务管理

Spring 通过 @Transactional 注解提供声明式事务，底层基于 AOP 实现。支持编程式事务（TransactionTemplate），但声明式更常用。

java 
@Transactional(
    propagation = Propagation.REQUIRED,      // 传播行为
    isolation   = Isolation.READ_COMMITTED,   // 隔离级别
    timeout     = 30,                          // 超时秒数
    rollbackFor = Exception.class             // 回滚策略
)
public void transfer(Long from, Long to, BigDecimal amount) {
    accountMapper.debit(from, amount);
    accountMapper.credit(to, amount);
}
⚠️
常见坑：@Transactional 仅对 public 方法生效；同一个类内方法互调不走代理，事务不生效；默认只回滚 RuntimeException 和 Error，受检异常不会回滚（需配置 rollbackFor）。

3 Spring Boot — 开箱即用的生产力引擎

Spring Boot 的核心哲学："约定大于配置"。它不是在 Spring Framework 上加功能，而是消除配置的繁琐，让你专注于业务代码。

⚙️ 自动配置（Auto-Configuration）

原理：通过 @EnableAutoConfiguration 和 spring.factories 文件，Spring Boot 根据 classpath 中的 jar 包自动推断你需要哪些 Bean 配置，省去大量 XML 和 Java Config。

java 
// 只需这几行，就是一个完整的 Web 应用
@SpringBootApplication  // = @Configuration + @EnableAutoConfiguration + @ComponentScan
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
💡
条件注解：自动配置依赖一系列 @ConditionalOnXxx 注解，如 @ConditionalOnClass（classpath 有某类才生效）、@ConditionalOnMissingBean（用户未自定义才生效）、@ConditionalOnProperty（配置项存在才生效）。可以用 --debug 查看哪些自动配置生效/未生效。

📦 起步依赖（Starter）

核心思想：一个 Starter 就是一个依赖包集合，引入它就等于引入了该功能所需的所有依赖，且版本经过测试兼容。告别版本冲突地狱。

📊 Actuator — 生产级运维端点

提供开箱即用的运维监控能力，包括健康检查、应用指标、环境信息、日志级别动态调整等。Spring Boot Admin 可以基于 Actuator 构建可视化监控面板。

端点功能示例
/actuator/health健康检查（含 DB、Redis 等组件状态）K8s liveness/readiness probe
/actuator/metrics应用指标（JVM 内存、HTTP 请求、线程池）接入 Prometheus + Grafana
/actuator/info自定义应用信息（版本、构建时间等）配合 CI/CD 注入版本号
/actuator/env环境变量 & 配置属性排查配置是否生效
/actuator/loggers动态调整日志级别线上调试，临时开启 DEBUG
/actuator/threaddump线程 dump排查死锁 / 线程池满

🔍 服务注册与发现

核心问题：微服务实例的 IP 和端口是动态变化的，需要一个注册中心来统一管理服务元数据。服务启动时注册，定时心跳续约，调用方从注册中心获取可用实例列表。

🇪

AP 模型（高可用 + 分区容错），自我保护机制。
已进入维护模式，存量项目仍在使用。

Nacos（Alibaba）

CP + AP 可切换，同时支持服务发现 + 配置中心，功能更全。
目前国内主流选择。

新项目首选

@SpringBootApplication

public class OrderApplication {

// 服务消费者 — 通过 Feign 声明式调用
@FeignClient("order-service")  // 服务名即注册中心的名称

📝 配置中心

核心问题：微服务数量多，配置散落在各服务中，修改一个配置要重启大量实例。配置中心统一管理所有配置，支持动态刷新——改完即生效，无需重启。

🛡️ 熔断器（Circuit Breaker）

核心问题：微服务间存在调用链，下游故障会级联上游，最终拖垮整个系统（雪崩效应）。熔断器在下游不可用时快速失败，保护上游服务。

🛡️

Sentinel（Alibaba）

流量控制 + 熔断降级 + 系统负载保护。
可视化控制台，实时监控，规则可动态调整。国内首选。

流量防护 / 熔断降级
⚡

Resilience4j

轻量级容错库，支持熔断、重试、限流、舱壁。
Spring Cloud 官方推荐，替代 Hystrix。

轻量级容错
java 
// Sentinel 熔断降级示例
@GetMapping("/orders/{id}")
@SentinelResource(
    value = "getOrder",
    blockHandler = "handleBlock",       // 限流/熔断触发
    fallback   = "handleFallback"       // 业务异常触发
)
public Order getOrder(@PathVariable Long id) {
    return orderService.findById(id);
}

// 降级处理方法
public Order handleFallback(Long id, Throwable t) {
    return "服务暂时不可用";
}

5 选型建议与总结

🌿 Spring 全家桶选型速查

  • 单体应用：Spring Boot + Spring MVC + MyBatis-Plus / JPA → 最简方案，快速交付
  • 微服务：Spring Cloud Alibaba（Nacos + Sentinel + Gateway）→ 国内生产首选，组件全且中文文档好
  • 轻量微服务：Spring Boot + Spring Cloud Gateway + Resilience4j → 如果团队对 Nacos 不熟，可用 Eureka / Consul 替代
  • 云原生：K8s + Spring Boot 即可，服务发现/配置管理/负载均衡可下沉到 K8s 层面
层级框架一句话总结
基础层 Spring Framework 打地基：IoC 容器 + AOP + 数据访问，是一切 Spring 应用的根基
开发层 Spring Boot 提效率：自动配置 + 起步依赖 + 内嵌服务器，让开发专注于业务
治理层 Spring Cloud 管集群：服务发现 + 配置中心 + 网关 + 熔断 + 链路追踪，微服务全家桶
