📋ArrayList 深度解析

底层结构

ArrayList 底层是动态数组。初始容量默认 10（首次 add 时创建）。容量不足时按 oldCapacity + oldCapacity >> 1（即 1.5 倍）扩容，通过 Arrays.copyOf 复制到新数组。

时间复杂度

操作时间复杂度说明
get(index)O(1)数组下标直接访问
add(E) 末尾追加均摊 O(1)不扩容时 O(1)，扩容时 O(n) 但均摊 O(1)
add(index, E) 指定位置插入O(n)需要移动后续元素
remove(index)O(n)需要移动后续元素
contains(O)O(n)线性遍历

代码示例

Java
import java.util.*;

// 创建 ArrayList
List<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
list.add("Go");
list.add("Rust");

// 索引访问 — O(1)
System.out.println(list.get(0));        // Java

// 指定位置插入 — O(n)
list.add(1, "C++");             // [Java, C++, Python, Go, Rust]

// 遍历
for (String s : list) {
    System.out.println(s);
}

// 函数式遍历 (Java 8+)
list.forEach(System.out::println);

// 删除元素
list.remove("Go");                 // 按对象删
list.remove(0);                    // 按索引删

// 指定初始容量 — 避免频繁扩容
List<Integer> bigList = new ArrayList<>(10000);
for (int i = 0; i < 10000; i++) {
    bigList.add(i);
}

// sublist — 视图操作，非拷贝
List<String> sub = list.subList(0, 2);
sub.set(0, "JavaScript");       // 原list也会变！

// 不可变 List (Java 9+)
List<String> immutable = List.of("A", "B", "C");
输出
Java
Java
C++
Python
Go
Rust
... (forEach 输出同上)
⚠️ subList 陷阱

subList 返回的是原 List 的视图，对 subList 的修改会影响原 List。同时，如果对原 List 做结构性修改（add/remove），subList 会抛出 ConcurrentModificationException。

🔗LinkedList 深度解析

底层结构

LinkedList 底层是双向链表，同时实现了 List 和 Deque 接口。每个节点（Node）包含 item、prev、next 三个字段。头尾各有一个哨兵引用。

时间复杂度

操作时间复杂度说明
get(index)O(n)需从头/尾遍历（会优化从近的一端开始）
addFirst / addLastO(1)直接操作头/尾指针
add(E) 末尾追加O(1)尾指针直接追加
add(index, E)O(n)需先遍历到指定位置
removeFirst / removeLastO(1)直接操作头/尾指针
contains(O)O(n)线性遍历

代码示例

Java
import java.util.*;

LinkedList<String> linked = new LinkedList<>();

// 链表特有操作 — O(1)
linked.addFirst("B");
linked.addFirst("A");          // [A, B]
linked.addLast("C");          // [A, B, C]

System.out.println(linked.getFirst());  // A
System.out.println(linked.getLast());   // C

linked.removeFirst();               // [B, C]

// 作为 Queue 使用
Queue<String> queue = new LinkedList<>();
queue.offer("task1");
queue.offer("task2");
System.out.println(queue.peek());   // task1 (查看队首)
System.out.println(queue.poll());   // task1 (取出并移除)

// 作为 Deque (双端队列) 使用
Deque<String> deque = new LinkedList<>();
deque.push("bottom");        // push = addFirst (栈操作)
deque.push("top");
System.out.println(deque.pop());    // top (LIFO)
输出
A
C
task1
task1
top

🔢HashSet 原理

底层结构

HashSet 底层是一个 HashMap，元素作为 key 存储，value 统一为一个 PRESENT 占位对象。利用 HashMap 的哈希桶实现 O(1) 的增删查。

核心逻辑：hashCode() 定位桶 → equals() 判断是否相同元素。

时间复杂度

操作平均最坏（哈希冲突）
add(E)O(1)O(n)
contains(O)O(1)O(n)
remove(O)O(1)O(n)

代码示例

Java
import java.util.*;

Set<String> set = new HashSet<>();
set.add("Java");
set.add("Python");
set.add("Java");          // 重复元素，不会被添加

System.out.println(set.size());      // 2
System.out.println(set.contains("Java")); // true

// 批量操作 — 交集、并集、差集
Set<Integer> setA = new HashSet<>(Set.of(1, 2, 3, 4));
Set<Integer> setB = new HashSet<>(Set.of(3, 4, 5, 6));

// 并集
Set<Integer> union = new HashSet<>(setA);
union.addAll(setB);
System.out.println(union);          // [1, 2, 3, 4, 5, 6]

// 交集
Set<Integer> inter = new HashSet<>(setA);
inter.retainAll(setB);
System.out.println(inter);          // [3, 4]

// 差集
Set<Integer> diff = new HashSet<>(setA);
diff.removeAll(setB);
System.out.println(diff);           // [1, 2]
输出
2
true
[1, 2, 3, 4, 5, 6]
[3, 4]
[1, 2]
⚠️ 自定义对象放入 HashSet

必须同时重写 hashCode() 和 equals()。如果两个对象 equals 为 true 但 hashCode 不同，HashSet 会将它们视为不同元素存入不同桶中——造成重复存储。

🌳TreeSet 原理

底层结构

TreeSet 底层是 TreeMap（红黑树）。元素按自然顺序（Comparable）或 Comparator 排序存储。适合需要排序、范围查询的场景。

时间复杂度

操作时间复杂度说明
add / contains / removeO(log n)红黑树查找路径
first / lastO(1)直接取最左/最右叶子
headSet / tailSet / subSetO(log n)范围视图

代码示例

Java
import java.util.*;

// 自然排序
TreeSet<Integer> tree = new TreeSet<>();
tree.add(30);
tree.add(10);
tree.add(20);
tree.add(50);
tree.add(40);

System.out.println(tree);           // [10, 20, 30, 40, 50] (自动排序)
System.out.println(tree.first());   // 10
System.out.println(tree.last());    // 50

// 范围查询
System.out.println(tree.headSet(30));   // [10, 20] (< 30)
System.out.println(tree.tailSet(30));   // [30, 40, 50] (>= 30)
System.out.println(tree.subSet(20, 50)); // [20, 30, 40]

// 自定义 Comparator — 降序
TreeSet<String> desc = new TreeSet<>(Comparator.reverseOrder());
desc.addAll(List.of("Banana", "Apple", "Cherry"));
System.out.println(desc);           // [Cherry, Banana, Apple]

// 导航方法
System.out.println(tree.ceiling(25));  // 30 (>= 25 的最小值)
System.out.println(tree.floor(25));    // 20 (<= 25 的最大值)
System.out.println(tree.higher(30));  // 40 (> 30 的最小值)
System.out.println(tree.lower(30));   // 20 (< 30 的最大值)

🗺️HashMap 深度解析

底层结构（JDK 1.8+）

HashMap 底层是 数组 + 链表 + 红黑树。每个数组位置称为「桶」（bucket）。

  • 初始容量 16，负载因子 0.75。元素数 > 容量 × 0.75 时触发扩容（翻倍）。
  • 桶内链表长度 ≥ 8 且数组容量 ≥ 64 → 链表转红黑树；树节点 ≤ 6 时退化为链表。
  • 定位桶：index = (n - 1) & hash(key)，其中 hash 是扰动后的 hashCode。

时间复杂度

操作平均最坏（哈希冲突严重）
put / get / removeO(1)O(log n) (红黑树)
containsKeyO(1)O(log n)
containsValueO(n)O(n)

代码示例

Java
import java.util.*;

Map<String, Integer> map = new HashMap<>();
map.put("Java", 1995);
map.put("Python", 1991);
map.put("Go", 2009);

// 基本操作
System.out.println(map.get("Java"));     // 1995
System.out.println(map.containsKey("Rust")); // false
System.out.println(map.size());           // 3

// getOrDefault — 避免 null 判断
int year = map.getOrDefault("Rust", -1);
System.out.println(year);                 // -1

// putIfAbsent — 仅在 key 不存在时插入
map.putIfAbsent("Java", 2000);  // 不会覆盖，Java 仍为 1995
map.putIfAbsent("Rust", 2010);  // 新增 Rust=2010

// merge — 合并值
map.merge("Java", 1, Integer::sum);  // Java=1996
map.merge("Kotlin", 2011, Integer::sum); // 新增 Kotlin=2011

// compute — 根据旧值计算新值
map.compute("Python", (k, v) -> v + 10);  // Python=2001

// 遍历方式 1: entrySet (最常用)
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " => " + entry.getValue());
}

// 遍历方式 2: forEach (Java 8+)
map.forEach((k, v) -> System.out.println(k + ": " + v));

// 按值排序
map.entrySet().stream()
    .sorted(Map.Entry.<String, Integer>comparingByValue())
    .forEach(e -> System.out.println(e.getKey() + "=" + e.getValue()));

// 不可变 Map (Java 9+)
Map<String, Integer> immutable = Map.of(
    "A", 1, "B", 2, "C", 3
);

⚠️ HashMap 非线程安全

多线程并发 put 可能导致数据丢失或死循环（JDK 1.7 链表头插法）。JDK 1.8 改为尾插法解决了死循环问题，但仍有数据丢失风险。多线程场景使用 ConcurrentHashMap。

🌲TreeMap 原理

底层结构

TreeMap 底层是 红黑树（自平衡二叉搜索树）。key 必须实现 Comparable 或提供 Comparator。适合需要按 key 排序或范围查找的场景。

代码示例

Java
import java.util.*;

TreeMap<String, Integer> treeMap = new TreeMap<>();
treeMap.put("banana", 3);
treeMap.put("apple", 5);
treeMap.put("cherry", 2);
treeMap.put("date", 4);

// 按 key 自然顺序排列
System.out.println(treeMap);  // {apple=5, banana=3, cherry=2, date=4}

// 范围查询
System.out.println(treeMap.headMap("cherry"));   // {apple=5, banana=3}
System.out.println(treeMap.tailMap("cherry"));   // {cherry=2, date=4}
System.out.println(treeMap.subMap("banana", "date")); // {banana=3, cherry=2}

// 导航方法
System.out.println(treeMap.firstKey());       // apple
System.out.println(treeMap.lastKey());        // date
System.out.println(treeMap.lowerKey("cherry")); // banana
System.out.println(treeMap.higherKey("cherry")); // date

LRU 缓存实现

LinkedHashMap 设置 accessOrder=true 并重写 removeEldestEntry 即可实现 LRU 缓存：

Java
import java.util.*;

// 10 容量的 LRU 缓存
Map<String, String> lru = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, String> eldest) {
        return size() > 10;
    }
};

🗺️ConcurrentHashMap 线程安全 Map

核心原理

版本实现方式锁粒度
JDK 1.7Segment[] 分段锁 + HashEntry[]Segment 级别（默认 16 段）
JDK 1.8+Node[] + CAS + synchronized桶级别（单个桶头节点）

JDK 1.8 实现：put 时先用 CAS 插入空桶；如果桶非空，synchronized 锁住桶头节点再插入/更新。读操作完全无锁（volatile 修饰 Node 的 val 和 next）。扩容时支持多线程协助迁移数据。

代码示例

Java
import java.util.concurrent.*;
import java.util.*;

// 创建 ConcurrentHashMap
ConcurrentHashMap<String, Integer> chm = new ConcurrentHashMap<>();

// 线程安全的 put — 内部 CAS + synchronized
chm.put("A", 1);
chm.put("B", 2);
chm.put("C", 3);

// 原子操作 — 比先 get 后 put 更安全高效
// putIfAbsent: 不存在才放入
chm.putIfAbsent("A", 100);     // A 仍为 1
chm.putIfAbsent("D", 4);       // D=4

// compute: 原子更新
chm.compute("A", (k, v) -> v + 10);   // A=11

// merge: 原子合并
chm.merge("B", 10, Integer::sum);    // B=12

// 多线程并发写入 — 完全安全
ExecutorService pool = Executors.newFixedThreadPool(3);
for (int i = 0; i < 1000; i++) {
    final int idx = i;
    pool.submit(() -> {
        chm.merge("counter", 1, Integer::sum);
    });
}
pool.shutdown();
pool.awaitTermination(2, TimeUnit.SECONDS);
System.out.println("counter = " + chm.get("counter"));  // 1000

// 遍历 — 弱一致性迭代器（不会抛 CME）
chm.forEach((k, v) -> System.out.println(k + "=" + v));

// 并行遍历 — 自动分片并行处理
chm.forEachEntry(2, e -> System.out.println(e.getKey()));
输出
counter = 1000
A=11
B=12
C=3
D=4
counter=1000
... (遍历顺序不固定)
💡 CHM vs HashMap + synchronized

CHM 锁粒度是单个桶，不同桶可完全并行读写。而 Collections.synchronizedMap 锁的是整个 Map 对象，性能差几个数量级。

📋CopyOnWriteArrayList 写时复制 List

核心原理

写操作（add/remove/set）时，先获取 ReentrantLock，然后复制整个底层数组到新数组进行修改，最后替换引用。读操作完全无锁，直接读取当前数组引用。

适用场景：读远多于写（如监听器列表、配置缓存），写操作极少但遍历时不能有并发修改。

Java
import java.util.concurrent.*;

CopyOnWriteArrayList<String> cowList = new CopyOnWriteArrayList<>();
cowList.add("listener1");
cowList.add("listener2");
cowList.add("listener3");

// 读操作 — 无锁，直接返回
System.out.println(cowList.get(0));    // listener1

// 遍历时写入 — 不会抛 ConcurrentModificationException
for (String s : cowList) {
    if (s.equals("listener2")) {
        cowList.add("listener4");  // 安全！但遍历的是旧快照
    }
    System.out.println(s);
}
System.out.println(cowList);  // [listener1, listener2, listener3, listener4]
输出
listener1
listener2
listener3
[listener1, listener2, listener3, listener4]
⚠️ 性能注意

每次写操作都复制整个数组，O(n) 时间和空间开销。如果写操作频繁，性能极差。数据量大时内存压力高（GC 负担）。

⏳CountDownLatch 一次性倒计时

核心原理

基于 AQS（AbstractQueuedSynchronizer）的共享模式。初始化一个 count，每次 countDown() 将 count 减 1，await() 阻塞到 count 归零。不可重置，是一次性的。

典型场景：主线程等待 N 个子线程完成后再继续。

Java
import java.util.concurrent.*;

// 3 个子任务
CountDownLatch latch = new CountDownLatch(3);

ExecutorService pool = Executors.newFixedThreadPool(3);

for (int i = 1; i <= 3; i++) {
    final int taskId = i;
    pool.submit(() -> {
        try {
            System.out.println("任务 " + taskId + " 开始执行");
            Thread.sleep(1000 * taskId);  // 模拟耗时
            System.out.println("任务 " + taskId + " 完成");
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            latch.countDown();  // 无论成功失败都要 countDown
        }
    });
}

// 主线程等待所有任务完成
System.out.println("主线程等待中...");
latch.await();
System.out.println("所有任务完成！");

pool.shutdown();
输出
主线程等待中...
任务 1 开始执行
任务 2 开始执行
任务 3 开始执行
任务 1 完成
任务 2 完成
任务 3 完成
所有任务完成！
⚠️ 必须在 finally 中 countDown

如果子线程异常退出但未 countDown，主线程将永久阻塞。最佳实践是在 finally 块中调用 latch.countDown()。

🔄CyclicBarrier 可循环屏障

核心原理

基于 ReentrantLock + Condition 实现。N 个线程调用 await() 后全部阻塞，当第 N 个线程到达时，屏障打开，所有线程同时被唤醒。可指定一个 barrierAction 在屏障打开时执行。可重置/重复使用（Cyclic 的含义）。

典型场景：多线程分阶段计算（如矩阵分块运算），每阶段所有线程都到齐后才能进入下一阶段。

Java
import java.util.concurrent.*;

// 3 个线程，屏障打开时执行汇总动作
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("--- 所有线程到达屏障，执行汇总 ---");
});

Runnable task = () -> {
    try {
        String name = Thread.currentThread().getName();
        System.out.println(name + " 阶段1完成");
        barrier.await();  // 等待其他线程

        System.out.println(name + " 阶段2完成");
        barrier.await();  // 第二次使用 — 可复用！

        System.out.println(name + " 全部完成");
    } catch (Exception e) {
        Thread.currentThread().interrupt();
    }
};

ExecutorService pool = Executors.newFixedThreadPool(3);
for (int i = 0; i < 3; i++) {
    pool.submit(task);
}
pool.shutdown();

🚦Semaphore 信号量 / 限流

核心原理

基于 AQS 共享模式。维护一组许可（permits），acquire() 获取许可（如果可用则立即返回，否则阻塞），release() 释放许可。许可数可以大于 1，实现同时允许多个线程访问。

典型场景：限流（如数据库连接池最多 10 个连接）、资源池控制。

Java
import java.util.concurrent.*;

// 只允许 3 个线程同时访问
Semaphore semaphore = new Semaphore(3);

ExecutorService pool = Executors.newFixedThreadPool(10);

for (int i = 0; i < 10; i++) {
    final int id = i;
    pool.submit(() -> {
        try {
            System.out.println("线程 " + id + " 等待许可...");
            semaphore.acquire();  // 获取许可（可能阻塞）
            System.out.println("线程 " + id + " 获得许可，开始工作");
            Thread.sleep(1000);  // 模拟工作
            System.out.println("线程 " + id + " 释放许可");
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } finally {
            semaphore.release();  // 释放许可
        }
    });
}

pool.shutdown();
pool.awaitTermination(10, TimeUnit.SECONDS);

🔐ReentrantLock 可重入锁

核心原理

基于 AQS 独占模式。支持公平锁（FIFO，先到先得）和非公平锁（默认，允许插队以提高吞吐量）。可重入：同一线程可多次获取同一把锁（state 递增），需等量释放。

比 synchronized 的优势：可中断（lockInterruptibly）、可超时（tryLock）、可尝试获取（tryLock 非阻塞）、支持多个 Condition。

Java
import java.util.concurrent.locks.*;

// 非公平锁（默认）
ReentrantLock lock = new ReentrantLock();
// 公平锁
ReentrantLock fairLock = new ReentrantLock(true);

// 标准用法 — try-finally 确保释放
lock.lock();
try {
    System.out.println("持有锁，执行临界区代码");
} finally {
    lock.unlock();  // 必须在 finally 中释放！
}

// tryLock — 非阻塞尝试
if (lock.tryLock()) {
    try {
        System.out.println("获取锁成功");
    } finally {
        lock.unlock();
    }
} else {
    System.out.println("获取锁失败，不阻塞");
}

// tryLock 带超时
if (lock.tryLock(3, TimeUnit.SECONDS)) {
    try {
        System.out.println("3秒内获取到锁");
    } finally {
        lock.unlock();
    }
}

// 可中断锁 — 响应 interrupt
try {
    lock.lockInterruptibly();
    try {
        System.out.println("获取到可中断锁");
    } finally {
        lock.unlock();
    }
} catch (InterruptedException e) {
    System.out.println("被中断，放弃获取锁");
}

// Condition — 精确唤醒
ReentrantLock prodLock = new ReentrantLock();
Condition notFull = prodLock.newCondition();
Condition notEmpty = prodLock.newCondition();

// 生产者-消费者模式
class BoundedBuffer<T> {
    private final Object[] items = new Object[10];
    private int putIdx, takeIdx, count;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();

    public void put(T item) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length)
                notFull.await();       // 满了就等待
            items[putIdx] = item;
            if (++putIdx == items.length) putIdx = 0;
            count++;
            notEmpty.signal();          // 通知消费者
        } finally {
            lock.unlock();
        }
    }

    @SuppressWarnings("unchecked")
    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0)
                notEmpty.await();       // 空了就等待
            T item = (T) items[takeIdx];
            items[takeIdx] = null;
            if (++takeIdx == items.length) takeIdx = 0;
            count--;
            notFull.signal();           // 通知生产者
            return item;
        } finally {
            lock.unlock();
        }
    }
}

Java
import java.util.concurrent.*;

// 手动创建线程池 — 不推荐用 Executors 工具方法
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    2,                          // corePoolSize
    4,                          // maximumPoolSize
    60L, TimeUnit.SECONDS,       // keepAliveTime
    new ArrayBlockingQueue<>(10),  // 有界队列，容量 10
    new ThreadFactory() {           // 自定义线程工厂
        private int count = 0;
        @Override
        public Thread newThread(Runnable r) {
            return new Thread(r, "my-pool-" + count++);
        }
    },
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
);

// 提交任务
for (int i = 0; i < 20; i++) {
    final int taskId = i;
    executor.submit(() -> {
        System.out.println(Thread.currentThread().getName()
            + " 执行任务 " + taskId);
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    });
}

// submit 返回 Future — 可获取结果

    Thread.sleep(1000);
    return "任务结果";
});

System.out.println(result);

// invokeAll — 批量提交并等待全部完成
List<Callable<Integer>> tasks = List.of(

    () -> { Thread.sleep(300); return 2; },
    () -> { Thread.sleep(800); return 3; }
);

}

// 优雅关闭

    executor.shutdownNow(); // 强制停止
}

⚛️原子类 CAS 无锁并发

核心原理

原子类基于 CAS（Compare-And-Swap） 乐观锁实现。CAS 是 CPU 原子指令，比较内存值与预期值，相等则更新。Java 通过 Unsafe.compareAndSwapXxx 调用 CPU 的 CMPXCHG 指令。

ABA 问题：值从 A→B→A，CAS 检测不到中间变化。解决方案：AtomicStampedReference（加版本号）。

类别代表类用途
基本类型AtomicInteger, AtomicLong, AtomicBoolean原子计数器
引用类型AtomicReference, AtomicStampedReference原子更新对象引用
数组类型AtomicIntegerArray, AtomicReferenceArray原子更新数组元素
字段更新器AtomicIntegerFieldUpdater原子更新 volatile 字段
累加器 (JDK 8+)LongAdder, LongAccumulator高并发计数（比 AtomicLong 快）

代码示例

Java
import java.util.concurrent.atomic.*;
import java.util.concurrent.*;

// AtomicInteger — 原子计数器
AtomicInteger counter = new AtomicInteger(0);

// 多线程自增 — 线程安全
ExecutorService pool = Executors.newFixedThreadPool(10);
for (int i = 0; i < 10000; i++) {
    pool.submit(() -> counter.incrementAndGet());
}
pool.shutdown();
pool.awaitTermination(2, TimeUnit.SECONDS);
System.out.println(counter.get());  // 10000

// CAS 手动操作
int oldValue = counter.get();
int newValue = oldValue + 100;
while (!counter.compareAndSet(oldValue, newValue)) {
    // CAS 失败说明有其他线程修改了值，重新读取
    oldValue = counter.get();
    newValue = oldValue + 100;
}

// getAndUpdate — 函数式更新
counter.getAndUpdate(x -> x * 2);
System.out.println(counter.get());

// AtomicReference — 原子更新对象引用
AtomicReference<String> ref = new AtomicReference<>("initial");
boolean success = ref.compareAndSet(initial", "updated");
System.out.println(ref.get());  // updated

// AtomicStampedReference — 解决 ABA 问题
AtomicStampedReference<String> stamped =
    new AtomicStampedReference<>("A", 1);  // 初始值 A，版本号 1

// 线程1: A → B
stamped.compareAndSet("A", "B", 1, 2);
// 线程2: B → A（ABA！）
stamped.compareAndSet("B", "A", 2, 3);

// 线程3: 以为还是 A 版本1，但实际版本是 3 — CAS 失败！
boolean changed = stamped.compareAndSet("A", "C", 1, 4);
System.out.println("CAS 结果: " + changed);  // false (检测到 ABA)

// LongAdder — 高并发场景比 AtomicLong 快
LongAdder adder = new LongAdder();
for (int i = 0; i < 100000; i++) {
    pool = Executors.newFixedThreadPool(10);
    pool.submit(() -> adder.increment());
}
// LongAdder 内部分 Cell[] 数组，各线程操作不同 Cell，减少竞争
System.out.println(adder.sum());  // 100000

💡 LongAdder vs AtomicLong

低并发下 AtomicLong 更好（无 Cell 数组开销）。高并发下 LongAdder 通过分段累加减少 CAS 竞争，吞吐量大幅提升。LongAdder 在 sum() 时有遍历 Cell 的开销，不适合高频读取。

# ── 自定义上下文管理器 ──
class Timer:
    def __enter__(self):
        import time
        self.start = time.perf_counter()
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        import time
        self.elapsed = time.perf_counter() - self.start
        print(f"耗时: {self.elapsed:.4f}s")
        return False  # 不吞异常

with Timer() as t:
    sum(range(1000000))
# 耗时: 0.0234s

# ── contextlib.contextmanager 装饰器方式 ──
from contextlib import contextmanager

@contextmanager
def open_db_connection(db_url):
    print(f"连接数据库: {db_url}")
    conn = {"url": db_url}  # 模拟连接
    try:
        yield conn  # with 块内拿到的对象
    finally:
        print("关闭数据库连接")

with open_db_connection("postgres://localhost") as conn:
    print(f"使用连接: {conn['url']}")

⚡异步编程 async / await

async
声明异步函数/协程
await
等待协程完成（暂停当前协程）
Python
import asyncio

# ── async def 定义协程函数 ──
async def fetch_data(url, delay):
    print(f"开始获取 {url}")
    await asyncio.sleep(delay)  # 非阻塞等待
    print(f"完成获取 {url}")
    return f"数据来自 {url}"

# ── 串行执行：总耗时 = 1+2+3 = 6s ──
async def main_serial():
    r1 = await fetch_data("api1", 1)
    r2 = await fetch_data("api2", 2)
    r3 = await fetch_data("api3", 3)
    return [r1, r2, r3]

# ── 并发执行：总耗时 = max(1,2,3) = 3s ──
async def main_concurrent():
    results = await asyncio.gather(
        fetch_data("api1", 1),
        fetch_data("api2", 2),
        fetch_data("api3", 3),
    )
    return results

# ── asyncio.create_task — 后台运行 ──
async def main_task():
    task = asyncio.create_task(fetch_data("bg", 2))
    print("任务已创建，做其他事...")
    await asyncio.sleep(1)
    print("等待任务完成...")
    result = await task
    print(f"结果: {result}")

# ── asyncio.wait_for — 超时控制 ──
async def main_timeout():
    try:
        result = await asyncio.wait_for(
            fetch_data("slow", 10),
            timeout=2
        )
    except asyncio.TimeoutError:
        print("请求超时！")

# 运行事件循环 (Python 3.7+)
print("=== 串行 ===")
asyncio.run(main_serial())

print("=== 并发 ===")
asyncio.run(main_concurrent())

print("=== 后台任务 ===")
asyncio.run(main_task())

print("=== 超时控制 ===")
asyncio.run(main_timeout())

💡 async/await 核心要点

async def 定义协程函数，调用它返回一个协程对象（不会立即执行）。await 只能在 async 函数内使用，暂停当前协程直到 awaitable 完成。asyncio.run() 启动事件循环。适合 IO 密集型（网络请求、文件读写），不适合 CPU 密集型。

Python
# ── and / or / not ──
print(True and False)   # False
print(True or False)    # True
print(not True)          # False

# 短路求值 — and 返回第一个假值，or 返回第一个真值
print(0 and "hello")       # 0 (第一个假值)
print("" and "world")      # ""
print(None or "default")    # default
print(0 or "" or None or "fallback")  # fallback

# 常用技巧：默认值
name = None
display_name = name or "匿名用户"
print(display_name)  # 匿名用户

# ── in: 成员判断 ──
print(3 in [1, 2, 3])           # True
print("a" in {"a": 1, "b": 2})    # True (检查 key)
print("py" in "python")          # True (子串判断)

# not in
print(5 not in [1, 2, 3])      # True

# ── is: 身份判断（同一对象） ──
a = [1, 2, 3]
b = a
c = [1, 2, 3]

print(a is b)      # True  (同一对象)
print(a is c)      # False (不同对象，值相同)
print(a == c)      # True  (值相等)

# is None — 判 None 的正确方式
x = None
print(x is None)   # True
print(x is not None)  # False

# ⚠️ 小整数缓存：-5 ~ 256 范围内 is 为 True
print(256 is 256)   # True  (缓存)
print(257 is 257)  # 可能 False (不缓存)

Java vs Python 关键字对比

☕ Java 关键字 (约 50 个)

有 final (常量/不可继承)、static、synchronized、volatile、native、abstract、interface、enum、instanceof、switch/case/default、do、new、this/super、throws/throw 等 Python 没有的关键字。

🐍 Python 关键字 (35 个)

有 yield (生成器)、async/await (协程原生)、with (上下文管理器)、nonlocal (闭包变量)、elif (缩写)、True/False/None (首字母大写) 等 Java 没有的关键字。Python 没有 switch (3.10+ 用 match/case)。
