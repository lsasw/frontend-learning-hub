LangChain 是什么？

LangChain 是一个构建 LLM 驱动应用的开源框架。 它把「调模型」这件事做了系统化封装：提示模板、链式调用、工具集成、记忆管理、RAG 检索——全部开箱即用。

案例一：最简 LLM 调用

三行代码让 GPT-4 帮你写估值分析报告开头。

Python 
from langchain.chat_models import ChatOpenAI
from langchain.schema import HumanMessage

# 初始化模型
llm = ChatOpenAI(model="gpt-4", temperature=0.7)

# 直接问
response = llm.invoke([
    HumanMessage(content="请用中文写一段贵州茅台 DCF 估值分析的开场白")
])

# response.content 就是答案
print(response.content)

# 输出示例:
# "贵州茅台作为中国白酒行业龙头，其护城河深度和现金流稳定性
#  使得 DCF 估值方法具有较高的适用性。本次分析基于以下核心假设..."
实战要点：temperature 控制创造性：0=严谨/金融分析，0.7=创意/文案；model 可按 Agent 分层（复杂推理用 GPT-4，简单格式化用 GPT-3.5）

案例二：Prompt 模板 + 链式调用

金融场景：自动生成多情景估值摘要。模板保证格式一致，Chain 负责串联。

Python 
from langchain.chat_models import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema import StrOutputParser

llm = ChatOpenAI(model="gpt-4", temperature=0)

# 定义模板 —— 输入变量用 { } 标记
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位资深估值分析师，输出使用中文。"),
    ("human", """请对 {company} 做 DCF 估值摘要：

核心假设：
- 预测期: {forecast_years} 年
- WACC: {wacc}%
- 永续增长率: {growth_rate}%

请输出：合理股价区间、关键变量敏感性、一句话结论。""")
])

# 构建 Chain: Prompt → LLM → 文本输出
chain = prompt | llm | StrOutputParser()

# 一口气跑不同公司
companies = [
    {"company": "宁德时代", "forecast_years": 10, "wacc": 8.5, "growth_rate": 3.0},
    {"company": "比亚迪",   "forecast_years": 10, "wacc": 9.0, "growth_rate": 3.5},
]

for params in companies:
    result = chain.invoke(params)
    print(f"\n=== {params['company']} ===\n{result}")

核心概念 LCEL (LangChain Expression Language)：用 | 管道符串联组件，比旧版 LLMChain 更简洁、支持流式、可组合。

案例三：RAG — 让 LLM 读你的研报库

问题：GPT 不知道你公司的内部估值模型参数。方案：把 PDF 研报向量化，让 LLM 检索后回答。

┌───────┐    ┌───────────┐    ┌───────────┐    ┌────────────┐
│ 用户   │──→│ 检索系统    │──→│ 增强 Prompt│──→│  LLM 回答   │
│ 提问   │    │ (向量检索)  │    │ (上下文注入)│    │            │
└───────┘    └───────────┘    └───────────┘    └────────────┘
                  ↑
          ┌──────────────┐
          │ 你的 PDF 研报 │ → Chunk → Embed → VectorStore
          └──────────────┘
Python 
from langchain.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma
from langchain.chat_models import ChatOpenAI
from langchain.chains import RetrievalQA

# 1. 加载 PDF 研报 → 切块
loader = PyPDFLoader("茅台2025年Q1业绩分析.pdf")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 2. 向量化 → 存入 Chroma
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)

# 3. 构建 RAG 检索链
qa_chain = RetrievalQA.from_chain_type(
    llm=ChatOpenAI(model="gpt-4"),
    chain_type="stuff",
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3})
)

# 4. 提问 —— LLM 会基于检索到的研报片段回答
answer = qa_chain.run("茅台Q1营收增速和毛利率是多少？")
print(answer)

# 也可以问跨文档问题：
answer2 = qa_chain.run("对比茅台、五粮液Q1的净利润率和增速")
避坑指南：chunk_size 要平衡，太大检索不准、太小语义丢失；embedding 模型推荐 text-embedding-3-small（性价比最高）；检索 top-k 建议 3~5 个；如果答案"跑偏"说明 chunk 切太碎。

案例四：Agent + Tool Calling — 让 AI 查实时股价

LangChain Agent 可以根据用户意图自动选择调用哪个工具（函数）。这里演示：AI 自动决定查股价还是算估值。

Python 
from langchain.chat_models import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain.tools import tool
from langchain.prompts import ChatPromptTemplate, MessagesPlaceholder

# ── 定义工具（普通函数 + @tool 装饰器）──
@tool
def get_stock_price(symbol: str) -> str:
    """获取A股实时股价。symbol: 股票代码如 '600519'（茅台）"""
    # 模拟数据，实际接东方财富 API
    prices = {"600519": "1680.00元", "000858": "142.50元", "300750": "218.00元"}
    return prices.get(symbol, "未找到该股票")

@tool
def quick_dcf(fcf: float, wacc: float, growth: float) -> str:
    """快速 DCF 估值。fcf: 自由现金流(亿), wacc: 折现率(%), growth: 永续增长率(%)"""
    terminal_value = fcf * (1 + growth/100) / (wacc/100 - growth/100)
    return f"永续终值: {terminal_value:.1f}亿"

# ── 构建 Agent ──
llm = ChatOpenAI(model="gpt-4")
tools = [get_stock_price, quick_dcf]
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是量化分析助手，能用工具查股价或快速估算。"),
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="agent_scratchpad"),
])
agent = create_openai_tools_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# Agent 会自动选工具！
executor.invoke({"input": "茅台现在多少钱？"})
# → 自动调用 get_stock_price(symbol="600519")

executor.invoke({"input": "宁德时代 WACC 8.5%、永续增长3%、自由现金流500亿，估一下"})
# → 自动调用 quick_dcf(fcf=500, wacc=8.5, growth=3)
设计原则：工具的 docstring 就是 Agent 的"使用说明书"——写清楚什么时候用、参数什么含义。如果不写 docstring，Agent 不知道这个工具能干嘛。

案例五：Memory — 让 AI 记住对话上下文

默认每次调用是"失忆"的。加了 Memory，AI 会带着之前的对话历史回答。

Python 
from langchain.chat_models import ChatOpenAI
from langchain.memory import ConversationBufferMemory
from langchain.chains import ConversationChain

llm = ChatOpenAI(model="gpt-4")
memory = ConversationBufferMemory()
chain = ConversationChain(llm=llm, memory=memory)

# 第一轮：设定上下文
chain.run("我叫七智，我重仓茅台成本1500元")
# AI: "你好七智！茅台1500建仓成本不错..."

# 第二轮：AI 记得上下文
chain.run("如果明天涨5%我浮盈多少？")
# AI: "按1500成本、涨5%到1575，每手浮盈7500元..."

# 没有 Memory 的话，第二轮AI会问"你是谁？什么成本？"
Memory 类型适用场景特点
ConversationBufferMemory短对话存全部历史，Token 消耗线性增长
ConversationSummaryMemory长对话对历史做摘要压缩，省 Token
ConversationBufferWindowMemory固定窗口只保留最近 K 轮，成本可控
VectorStoreRetrieverMemory知识持久化向量检索相关历史片段

LangGraph 是什么？

LangGraph 是 LangChain 生态的状态机 + 图编排引擎。 如果说 LangChain 解决"怎么调 LLM"，LangGraph 解决"怎么控制多个 LLM 调用的流向和循环"。

核心概念

  • State（状态）：在整个图执行过程中传递的共享字典
  • Node（节点）：执行单元，可以是 LLM 调用、工具调用、条件判断
  • Edge（边）：定义节点间的流转方向
  • Conditional Edge（条件边）：根据状态动态决定下一跳

相比 LangChain Agent 的优势

  • 更可控：图的每个分叉都是你显式定义的
  • 更透明：执行路径可追踪，方便调试
  • 支持循环：Agent 可以反复思考 → 调用工具 → 检查 → 再思考
  • 原生支持 Human-in-the-Loop：可中断等人工审批

案例一：DCF 估值多步骤状态机

金融实战：把 DCF 估值拆成 4 步 → 采集数据 → 计算 WACC → 跑 DCF → 输出 Excel，LangGraph 管理流转。

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  collect  │──→│calc_wacc │──→│ run_dcf  │──→│  export  │
│ 采集数据   │    │ 计算WACC │    │ DCF建模  │    │ Excel导出│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
       ↑                                              │
       └──────────── 数据不全，回头重采 ───────────────┘
Python 
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, END

# 1. 定义 State —— 图里流转的共享数据结构
class DCFState(TypedDict):
    ticker: str              # 股票代码
    fcf: float             # 自由现金流
    wacc: float            # 加权平均资本成本
    growth_rate: float     # 永续增长率
    intrinsic_value: float  # 内在价值
    report: str            # 最终报告

# 2. 定义每个 Node（节点=执行步骤）
def collect_data(state: DCFState) -> DCFState:
    """从数据库/API 采集财务数据"""
    # 模拟采集结果
    state["fcf"] = 500.0   # 亿
    state["wacc"] = 0.085  # 8.5%
    state["growth_rate"] = 0.03  # 3%
    return state

def calculate_wacc(state: DCFState) -> DCFState:
    """计算/校验 WACC"""
    # 这里可以调用 LLM 做参数合理性检查
    if state["wacc"] < 0.05 or state["wacc"] > 0.15:
        state["wacc"] = 0.09  # 默认合理值
    return state

def run_dcf(state: DCFState) -> DCFState:
    """执行 DCF 计算"""
    fcf, wacc, g = state["fcf"], state["wacc"], state["growth_rate"]
    terminal = fcf * (1 + g) / (wacc - g)
    state["intrinsic_value"] = round(terminal, 2)
    return state

def export_report(state: DCFState) -> DCFState:
    """生成报告"""
    state["report"] = (
        f"【{state['ticker']} DCF估值报告】\n"
        f"自由现金流: {state['fcf']}亿\n"
        f"WACC: {state['wacc']*100:.1f}%\n"
        f"内在价值: {state['intrinsic_value']}亿"
    )
    return state

# 3. 构建图
graph = StateGraph(DCFState)
graph.add_node("collect", collect_data)
graph.add_node("calc_wacc", calculate_wacc)
graph.add_node("run_dcf", run_dcf)
graph.add_node("export", export_report)

# 4. 定义边（流转方向）
graph.set_entry_point("collect")
graph.add_edge("collect", "calc_wacc")
graph.add_edge("calc_wacc", "run_dcf")
graph.add_edge("run_dcf", "export")
graph.add_edge("export", END)

# 5. 编译 & 运行
app = graph.compile()
result = app.invoke({"ticker": "600519"})
print(result["report"])

# 输出:
# 【600519 DCF估值报告】
# 自由现金流: 500.0亿
# WACC: 8.5%
# 内在价值: 9363.64亿

案例二：Agent 自主循环 — 思考→行动→观察

这是 LangGraph 最经典的 ReAct 模式：Agent 反复在 Think（思考）→ Act（调工具）→ Observe（看结果）之间循环，直到得出结论。

        ┌──────────────────────────────┐
        │         ReAct Agent           │
        │                               │
        │   ┌──────┐   ┌──────┐        │
   START──→│ agent │──→│tools?│──YES──┐ │
        │   └──────┘   └──────┘       │ │
        │       ↑          │ NO        │ │
        │       │          ↓           │ │
        │       └──── END ←┘          │ │
        │               ↑              │ │
        │   ┌──────┐    │              │ │
        │   │tools │────┘              │ │
        │   └──────┘                   │ │
        └──────────────────────────────┘
Python 
from typing import TypedDict, Annotated, Literal
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
from langchain.chat_models import ChatOpenAI
from langchain.tools import tool
from langchain.schema import HumanMessage
import json

# ── 工具 ──
@tool
def get_balance_sheet(ticker: str) -> str:
    """获取资产负债表摘要。ticker: 股票代码"""
    data = {"600519": {"现金": "1200亿", "总负债": "350亿", "ROE": "32%"}}
    return json.dumps(data.get(ticker, {}), ensure_ascii=False)

@tool
def get_industry_pe(industry: str) -> str:
    """获取行业平均PE。industry: 行业名称"""
    pe = {"白酒": "25.3x", "新能源": "35.7x", "银行": "5.8x"}
    return pe.get(industry, "暂无数据")

tools = [get_balance_sheet, get_industry_pe]
llm = ChatOpenAI(model="gpt-4").bind_tools(tools)

# ── State ──
class AgentState(TypedDict):
    messages: list

# ── Node ──
def call_agent(state: AgentState):
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

# 条件路由: 如果需要调工具 → tools 节点; 否则 → END
def should_continue(state: AgentState) -> Literal["tools", "__end__"]:
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"
    return "__end__"

# ── 构建图 ──
workflow = StateGraph(AgentState)
workflow.add_node("agent", call_agent)
workflow.add_node("tools", ToolNode(tools))
workflow.set_entry_point("agent")
workflow.add_conditional_edges("agent", should_continue)
workflow.add_edge("tools", "agent")  # 工具结果返回给agent重新思考

app = workflow.compile()

# 问一个问题，Agent 会自动查工具 → 思考 → 回答
result = app.invoke({
    "messages": [HumanMessage(content="茅台资产负债表怎么样？白酒行业PE多少？")]
})
print(result["messages"][-1].content)

案例三：并行分支 — 同时分析多家公司

LangGraph 支持并行执行：同时跑茅台、五粮液、宁德的估值，结果汇总比较。

                    ┌─────────────┐
                    │  split_tasks │ (分发)
                    └──┬───┬───┬──┘
                       │   │   │
                  ┌────↓ ┌─↓ ┌─↓────┐
                  │茅台  │五粮│ 宁德  │  (三个节点并行)
                  └──┬───┴─┬─┴──┬───┘

from langgraph.graph import StateGraph, END
from typing import TypedDict
import operator

class ParallelState(TypedDict):
    tickers: list
    results: dict   # {ticker: value}
    comparison: str

# 并行节点: 每个 ticker 独立估值
def valuation_node(state: ParallelState, ticker: str) -> dict:
    # 模拟: 实际可以调 LLM 或 API
    values = {"600519": 1680, "000858": 142, "300750": 218}
    return {"results": {ticker: values.get(ticker, 0)}}

def compare_results(state: ParallelState) -> ParallelState:
    r = state["results"]
    best = max(r, key=r.get)
    state["comparison"] = (
        f"估值对比: 茅台 {r.get('600519')}元 | "
        f"五粮液 {r.get('000858')}元 | "
        f"宁德时代 {r.get('300750')}元\n"
        f"股价最高: {best}"
    )
    return state

# 构建并行图（简化示意，实际需要 Send API）
# LangGraph 的 Send API 可以实现动态并行 fan-out
# 更多详见: langgraph docs / how-tos / map-reduce
提示：LangGraph 的 Send API 支持动态并行分发（fan-out）→ 并行执行 → 汇聚（fan-in），非常适合批量估值、批量报告生成等场景。

案例四：Human-in-the-Loop — 估值结果人工审批

金融场景的核心需求：AI 算完估值不是直接输出，而是先"暂停等审批"。

Python 
from langgraph.graph import StateGraph, END
from langgraph.checkpoint import MemorySaver
from typing import TypedDict

class ApprovalState(TypedDict):
    valuation: dict
    status: str        # "pending" | "approved" | "rejected"
    final_report: str

def do_valuation(state: ApprovalState) -> ApprovalState:
    state["valuation"] = {"price": 1680, "upside": "+12%", "risk": "中等"}
    state["status"] = "pending"
    return state

# ── 中断节点: 执行到这里会暂停，等人类批准 ──
def human_approval(state: ApprovalState) -> ApprovalState:
    """此处由人类决定 approve/reject"""
    return state

def on_approved(state: ApprovalState) -> ApprovalState:
    state["final_report"] = f"报告已生成。估值: {state['valuation']}"
    return state

def on_rejected(state: ApprovalState) -> ApprovalState:
    state["final_report"] = "审批未通过。请调整参数后重新估值。"
    return state

# 构建审批图
workflow = StateGraph(ApprovalState)
workflow.add_node("valuation", do_valuation)
workflow.add_node("approval", human_approval)
workflow.add_node("approved", on_approved)
workflow.add_node("rejected", on_rejected)

workflow.set_entry_point("valuation")
workflow.add_edge("valuation", "approval")
workflow.add_conditional_edges("approval",
    lambda s: "approved" if s["status"]=="approved" else "rejected",
    {"approved": "approved", "rejected": "rejected"}
)
workflow.add_edge("approved", END)
workflow.add_edge("rejected", END)

# interrupt_before=["approval"] → 在 approval 之前暂停
app = workflow.compile(
    checkpointer=MemorySaver(),
    interrupt_before=["approval"]
)

# 第一轮: 执行到 approval 前自动暂停
config = {"configurable": {"thread_id": "1"}}
result = app.invoke({}, config)
print(result["status"])  # "pending" — 等待人类点击

# 人类审核后继续:
app.update_state(config, {"status": "approved"})
final = app.invoke(None, config)
print(final["final_report"])

Python — AI 开发的基石

上面 LangChain 和 LangGraph 全是用 Python 写的。掌握 Python 的异步、类型注解、装饰器、Pydantic 数据模型 是写好 AI 应用的前提。以下聚焦 AI 开发中最常用的 6 个核心技能。

案例一：asyncio — AI 调用的生命线

LLM API 调用是 I/O 密集型（等网络返回），必须用异步才能不阻塞。LangChain/LangGraph 内部全是 async。

Python 
import asyncio

# 同步写法（串行，慢）—— 不推荐
# r1 = llm.invoke("问题1")  # 等3秒
# r2 = llm.invoke("问题2")  # 又等3秒 → 总计6秒

# 异步写法（并发，快）—— 推荐
async def batch_analyze(tickers):
    """同时分析多只股票，互不阻塞"""
    tasks = []
    for t in tickers:
        # 创建协程任务
        task = analyze_one_stock(t)
        tasks.append(task)

    # 全部并发执行！3只股票也是3秒，不是9秒
    results = await asyncio.gather(*tasks)
    return results

async def analyze_one_stock(ticker: str) -> dict:
    """模拟 LLM 调用（实际替换为 llm.ainvoke）"""
    await asyncio.sleep(1)  # 模拟网络延迟
    return {"ticker": ticker, "score": "A"}

# 运行
result = asyncio.run(batch_analyze(["600519", "000858", "300750"]))
print(result)
# 输出: 3条结果，总耗时约1秒（不是3秒）
记住：所有 LangChain 方法都有 async 版本——llm.ainvoke()、chain.ainvoke()、graph.ainvoke()。写 AI 应用，凡是 I/O 都用 async。

案例二：Pydantic — AI 输出的结构化保证

LLM 返回的是自然语言文本，但你的程序需要结构化数据。Pydantic 是 LangChain/LangGraph 的默认数据验证层。

Python 
from pydantic import BaseModel, Field
from typing import List, Optional
from langchain.chat_models import ChatOpenAI
from langchain.prompts import ChatPromptTemplate

# 1. 定义输出结构 — 这就是你和LLM的"合同"
class StockAnalysis(BaseModel):
    ticker: str = Field(description="股票代码")
    valuation: str = Field(description="估值判断: 低估/合理/高估")
    fair_price: float = Field(description="合理股价")
    confidence: float = Field(ge=0, le=1, description="置信度 0~1")
    key_risks: List[str] = Field(description="核心风险列表")
    summary: str = Field(max_length=200, description="一句话总结")

# 2. 让 LLM 直接输出这个结构
llm = ChatOpenAI(model="gpt-4")
structured_llm = llm.with_structured_output(StockAnalysis)

# 3. 调用 — 返回的是 Pydantic 对象，不是字符串！
result = structured_llm.invoke("分析茅台当前的估值水平")

# 类型安全: IDE 可以自动补全 result.ticker, result.fair_price
print(f"股票: {result.ticker}")
print(f"估值: {result.valuation}")
print(f"合理价: {result.fair_price}")
print(f"置信度: {result.confidence:.0%}")
print(f"风险: {', '.join(result.key_risks)}")

# 4. 一键转 dict/JSON，直接存数据库
data = result.model_dump()
# {"ticker": "600519", "valuation": "合理", "fair_price": 1700.0, ...}
关键技巧：用 with_structured_output() 让 LLM 直接返回 Pydantic 对象，省去正则解析的麻烦。Field 的 description 会被注入到 Prompt 里指导 LLM 输出格式。

案例三：装饰器 — LangChain Tool 的定义核心

LangChain 的 @tool 装饰器把普通函数变成 Agent 可调用的工具，背后的原理就是 Python 装饰器。

Python 
import time
from functools import wraps

# ── 理解装饰器本质 ──
@decorator_name
def func(): ...
# 等价于: func = decorator_name(func)

# ── 实战: 给估值函数加日志和计时 ──
def log_execution(func):
    """装饰器: 自动记录任何函数的执行日志"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        print(f"[LOG] 开始执行 {func.__name__}")
        try:
            result = func(*args, **kwargs)
            elapsed = time.time() - start
            print(f"[LOG] {func.__name__} 完成, 耗时 {elapsed:.2f}s")
            return result
        except Exception as e:
            print(f"[ERROR] {func.__name__} 失败: {e}")
            raise
    return wrapper

# 使用装饰器 — 零侵入，函数本身不改一行
@log_execution
def dcf_valuation(fcf, wacc, growth):
    terminal = fcf * (1 + growth) / (wacc - growth)
    return terminal

@log_execution
def pe_valuation(earnings, pe):
    return earnings * pe

# 调用 — 自动打印日志
v1 = dcf_valuation(500, 0.085, 0.03)
v2 = pe_valuation(80, 25)

# [LOG] 开始执行 dcf_valuation
# [LOG] dcf_valuation 完成, 耗时 0.00s
# [LOG] 开始执行 pe_valuation
# [LOG] pe_valuation 完成, 耗时 0.00s

案例四：yield — LLM 流式输出的背后

ChatGPT 那种一个字一个字往外蹦的效果，底层就是 Python 生成器 + Streaming。LangChain 的 .stream() 方法返回的就是生成器。

Python 
from langchain.chat_models import ChatOpenAI

llm = ChatOpenAI(model="gpt-4", streaming=True)

# 流式输出——一个字一个字打印，像ChatGPT
for chunk in llm.stream("用200字概括DCF估值法的核心逻辑"):
    print(chunk.content, end="", flush=True)

# ── 理解 yield 本质 ──
def read_large_portfolio(filepath):
    """用生成器逐行读取大型持仓文件，不爆内存"""
    with open(filepath, "r") as f:
        for line in f:
            yield line.strip().split(",")

# 10GB 持仓文件也能处理——每次只加载一行到内存
for ticker, shares, cost in read_large_portfolio("portfolio_10gb.csv"):
    process_one_stock(ticker, shares, cost)

案例五：Type Hints — LangGraph State 的基石

LangGraph 的 State 定义全部依赖 TypedDict + 类型注解。写得越好，IDE 提示越好，bug 越少。

Python 
from typing import TypedDict, List, Optional, Literal, Union
from typing import Annotated    # LangGraph State 的核心!
from langgraph.graph import StateGraph
import operator

# ── 最佳实践: TypedDict 定义 LangGraph State ──
class ValuationState(TypedDict):

    model_type: Literal["DCF", "PE", "DDM"]  # 限定可选值

案例六：上下文管理器 — 资源安全与实验追踪

AI 开发中最常见的坑：数据库连接没关、文件句柄泄露、向量库没释放。with 语句保平安。

Python 
from contextlib import contextmanager
import time

# ── 实战: 估值实验上下文管理器 ──
@contextmanager
def valuation_experiment(name: str):
    """自动计时 + 错误捕获的估值实验包装器"""
    start = time.time()
    print(f"\n {'='*40}")
    print(f" 实验: {name}")
    print(f" {'='*40}")
    try:
        yield  # ← 这里执行 with 块里的代码
        elapsed = time.time() - start
        print(f"   状态: 成功 ✅ | 耗时: {elapsed:.2f}s")
    except Exception as e:
        elapsed = time.time() - start
        print(f"   状态: 失败 ❌ | 耗时: {elapsed:.2f}s | 错误: {e}")

# 用法 —— with 块结束自动打印时间
with valuation_experiment("茅台 DCF WACC=8.5%"):
    result = dcf_valuation(500, 0.085, 0.03)

with valuation_experiment("宁德 DCF WACC=10%"):
    result = dcf_valuation(400, 0.10, 0.035)

# 输出:
# ========================================
#  实验: 茅台 DCF WACC=8.5%
# ========================================
#    状态: 成功 ✅ | 耗时: 0.00s
# ========================================
#  实验: 宁德 DCF WACC=10%
# ========================================
#    状态: 成功 ✅ | 耗时: 0.00s
