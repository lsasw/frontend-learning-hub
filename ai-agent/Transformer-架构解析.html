Transformer 架构深度解析

「Attention Is All You Need」—— 告别 RNN 的顺序依赖，用纯粹的自注意力机制 重新定义序列建模。从数学原理到工程实现，全面理解这一改变 NLP 格局的里程碑架构。

1

背景与动机：为什么需要 Transformer

RNN / LSTM 的困境

在 Transformer 出现之前，序列建模（机器翻译、文本生成等）几乎被 RNN 和 LSTM 统治。但这些架构存在三个根本性问题：

  • 顺序计算瓶颈：RNN 每一步依赖上一步的隐藏状态，无法并行计算。长序列（比如 512 个 token）要串行走 512 步。
  • 长距离依赖衰减：即使 LSTM 有门控机制，信息在超过 30-50 步后仍然严重衰减，无法有效捕捉远距离的语义关系。
  • 梯度消失/爆炸：BPTT（沿时间反向传播）在长序列上不稳定，训练困难。
💡
Transformer 的核心思想：抛弃循环结构，用 Self-Attention 让序列中任意两个位置的 token 直接交互，计算复杂度从 O(n) 的串行步数变为 O(1) 的网络深度。

论文背景

2017 年 6 月，Google Brain 团队在论文 「Attention Is All You Need」 中首次提出 Transformer 架构。论文有 8 位作者，其中 Ashish Vaswani 和 Noam Shazeer 后分别创立了 Adept AI 和 Character.AI。

这篇论文的引用量在 2024 年已突破 12 万次，是所有 AI 论文中被引最多的之一，深远影响了 BERT、GPT、T5、Vision Transformer 等后续所有重要模型。

整体架构：Encoder-Decoder 框架

一图胜千言

六个关键组件

  1. Input / Output Embedding：将离散 token 映射到 d_model 维连续向量空间。
  2. Positional Encoding：注入位置信息，让模型感知 token 的顺序。
  3. Multi-Head Self-Attention：核心创新，并行计算序列中所有 token 之间的关注权重。
  4. Feed-Forward Network (FFN)：每个位置的独立非线性变换，增强表达能力。
  5. Add & Norm：残差连接 + 层归一化，稳定深层网络的训练。
  6. Masked Attention (Decoder)：因果掩码确保解码时只看过去、不看未来。
📐
标准 Transformer-base 配置：N = 6 层 Encoder + 6 层 Decoder，d_model = 512，h = 8 个注意力头，FFN 内层 d_ff = 2048。Transformer-big 则将层数翻倍。
3

自注意力机制 (Self-Attention)

直观理解

Self-Attention 让序列中的每个 token "看到"所有其他 token，并自己学会哪些 token 对当前 token 更重要。类比阅读一段话时，人的目光会在不同词之间跳跃，根据语义关联度分配注意力。

Scaled Dot-Product Attention：三步推导

给定输入序列 X ∈ Rn×d（n 个 token，每个 d 维），注意力计算分三步：

Step 1 — 生成 Q、K、V

通过三个可学习的线性变换矩阵将每个 token 投影到查询（Query）、键（Key）、值（Value）三个空间：

Q = X · WQ   K = X · WK   V = X · WV

其中 WQ, WK, WV ∈ Rd × d_k，d_k = d_model / h（如 512 / 8 = 64）。

Step 2 — 计算注意力分数

Q 与 K 做点积得到任一对 token 之间的「亲和度」分数，除以 √d_k 防止 softmax 进入饱和区：

Scores = (Q · KT) / √d_k

Step 3 — Softmax 归一化 + 加权求和

Attention(Q, K, V) = softmax( Q·KT / √d_k ) · V

多头注意力 (Multi-Head Attention)

为什么需要"多头"？

单个注意力头可能只关注某一种关系模式（比如「主谓关系」），但自然语言中存在多种并行的依赖关系。Multi-Head 的做法是：

  • 将 Q、K、V 拆分到 h 个独立子空间（每头维度 d_k = d_model / h）
  • 每个头独立计算 Scaled Dot-Product Attention
  • 将所有头的输出拼接后，再经过一次线性变换
MultiHead(Q,K,V) = Concat(head1, ..., headh) · WO
headi = Attention(Q·WiQ, K·WiK, V·WiV)

位置编码 (Positional Encoding)

为什么需要位置编码

Self-Attention 本身是 置换等变 (permutation-equivariant) 的 —— 打乱输入序列的 token 顺序，输出只是对应位置的排列发生变化，模型并不知道 token 的先后关系。然而自然语言中，语序至关重要：「狗咬人」和「人咬狗」完全不是一个意思。

Sinusoidal Positional Encoding（原始论文方案）

对位置 pos 和维度 i，注入三角函数编码：

PE(pos, 2i) = sin( pos / 100002i/d_model )
PE(pos, 2i+1) = cos( pos / 100002i/d_model )

为什么用 sin/cos？ 因为 sin(α+β) = sinα·cosβ + cosα·sinβ，这意味着位置 pos+k 的编码可以由位置 pos 的编码线性表出，模型天然具有捕捉相对位置的能力。

⚠️
实践中的分歧：原始论文使用固定的 Sinusoidal PE；BERT 和 GPT 系列则普遍使用 可学习的 Position Embedding（一个 (max_len × d_model) 的可训练矩阵）。两者效果相当，但可学习编码在实际工程中更灵活（如截断长度变化）。

前馈网络与残差连接

Position-wise FFN

每个 token 位置独立应用同一个两层全连接网络：

FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂

🔹 内层维度 d_ff = 2048（4× 扩展），外层压缩回 d_model = 512

🔹 等价于两个 1×1 卷积，现代实现常用 GELU 替代 ReLU

Add & Norm

每个子层后执行：

Output = LayerNorm(x + Sublayer(x))

🔹 残差连接（Add）：原始论文将 Add 放在 Norm 之前 (Post-LN)，缓解梯度消失

🔹 层归一化（Norm）：对每个样本的 d_model 维度做标准化

🔹 现代 LLM (GPT 系列) 多用 Pre-LN（先 LN 再子层），训练更稳定

✅
Pre-LN vs Post-LN：原始论文的 Post-LN 在深层网络中梯度容易爆炸，需要 warmup。Pre-LN 将 LayerNorm 放在子层之前（x + Sublayer(LayerNorm(x))），消除了 warmup 需求，是 GPT-2/3/4 的默认选择。

Masked Attention：解码器的因果约束

为什么 Decoder 需要 Mask

在训练时，Decoder 一次拿到完整的 target 序列。如果不加 mask，Decoder 的自注意力层会在预测第 i 个 token 时"偷看"第 i+1 个及之后的 token，这相当于考试时提前看答案。Masked Attention 的做法是：将注意力分数矩阵中未来位置的分数设为 -∞，softmax 后这些位置的权重归零。

训练与推理流程

训练阶段

  • Teacher Forcing：Decoder 输入是真实的 target 序列（右移一位），而非自回归生成的结果。
  • 并行计算：Encoder 和 Decoder 各自内部完全并行 —— 这比 RNN 快 10-100 倍。
  • 损失函数：交叉熵损失，在 Softmax 输出和目标 token 的 one-hot 之间计算。
  • 优化器：原始论文使用 Adam，配合自定义学习率调度（warmup_steps = 4000）。
  • Label Smoothing：ε = 0.1，防止模型过度自信，提升 BLEU 分数。

推理（自回归生成）

  • 逐 token 生成：Decoder 每次只生成一个 token，然后将其拼回输入，继续生成下一个。
  • KV Cache：缓存已计算的 Key 和 Value，避免每步重新计算历史 token 的注意力 —— 这是使推理可行的关键优化。
  • 停止条件：生成 EOS（End of Sequence）token 或达到最大长度。
  • 采样策略：贪心解码 / Beam Search / Top-k / Top-p (nucleus) / Temperature 控制。
⚡
KV Cache 是 Transformer 推理的核心优化：没有它，每个推理步的复杂度是 O(L²)，有它之后降为 O(L)。这就是为什么 GPT 系列在推理时，随着上下文变长，速度会线性下降而非平方下降。

变体与演进：从 Transformer 到现代 LLM

三大流派

Encoder-Only：BERT 系

  • 双向注意力：每个 token 能看到前后文
  • 预训练任务：MLM (Masked Language Modeling) + NSP
  • 代表：BERT (2018) → RoBERTa → DeBERTa → ELECTRA
  • 适用：文本分类、NER、问答等理解任务
  • 局限：不能直接做文本生成

Decoder-Only：GPT 系

  • 因果注意力：只能看过去的 token
  • 预训练任务：Next Token Prediction (自回归语言模型)
  • 代表：GPT-1 → GPT-2 → GPT-3 → GPT-4 → LLaMA → Claude
  • 适用：文本生成、对话、代码补全等生成式任务
  • 趋势：Scaling Law 驱动，规模越大能力越涌现

跨模态扩展

模型模态核心思想
Vision Transformer (ViT)图像将图像切为 16×16 patches，当作 token 序列输入 Transformer
CLIP图文双塔 Transformer 分别编码图文，对比学习对齐
Whisper语音音频 → 梅尔频谱 → Encoder → Decoder → 文本

核心代码示例 (PyTorch)

Scaled Dot-Product Attention

PYTHON
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q, K, V: (batch, heads, seq_len, d_k)
    mask: (batch, 1, seq_len, seq_len) or None
    """
    d_k = Q.size(-1)
    # 1. 计算注意力分数
    scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(
        torch.tensor(d_k, dtype=torch.float32)
    )
    # 2. 应用 mask（Decoder 用）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    # 3. Softmax + 加权求和
    attn_weights = F.softmax(scores, dim=-1)
    output = torch.matmul(attn_weights, V)
    return output, attn_weights

Multi-Head Attention

PYTHON
class MultiHeadAttention(torch.nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_k = d_model // num_heads
        self.num_heads = num_heads

        # 三个投影矩阵合并为一个大矩阵，GPU 更高效
        self.W_qkv = torch.nn.Linear(d_model, 3 * d_model)
        self.W_o = torch.nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, N, D = x.shape
        # 生成 Q, K, V 并拆分为多头
        qkv = self.W_qkv(x).reshape(B, N, 3, self.num_heads, self.d_k)
        qkv = qkv.permute(2, 0, 3, 1, 4)  # (3, B, H, N, d_k)
        Q, K, V = qkv[0], qkv[1], qkv[2]

        # 计算 Scaled Dot-Product Attention
        attn_out, _ = scaled_dot_product_attention(Q, K, V, mask)

        # 拼接多头并投影
        attn_out = attn_out.transpose(1, 2).reshape(B, N, D)
        return self.W_o(attn_out)

Transformer Encoder Layer

PYTHON
class TransformerEncoderLayer(torch.nn.Module):
    def __init__(self, d_model=512, num_heads=8, d_ff=2048, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        self.ffn = torch.nn.Sequential(
            torch.nn.Linear(d_model, d_ff),
            torch.nn.ReLU(),
            torch.nn.Dropout(dropout),
            torch.nn.Linear(d_ff, d_model),
        )
        self.norm1 = torch.nn.LayerNorm(d_model)
        self.norm2 = torch.nn.LayerNorm(d_model)
        self.dropout = torch.nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Self-Attention + Add & Norm
        attn_out = self.self_attn(x, mask)
        x = self.norm1(x + self.dropout(attn_out))
        # FFN + Add & Norm
        ffn_out = self.ffn(x)
        x = self.norm2(x + self.dropout(ffn_out))
        return x
