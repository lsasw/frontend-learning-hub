Dify — 为 AI 而生的应用构建平台

Dify 是一个开源 LLM 应用开发平台，将模型管理、Prompt 调试、RAG 检索、工作流编排、Agent 能力全部集成在可视化 Web 界面中。如果说 LangChain 是给程序员用的代码库，那 Dify 就是给所有人用的 AI 工厂。

🧩 应用层

Chatflow · Workflow · Agent · Text Generator

⚙️ 编排层

DAG 工作流引擎 · 条件分支 · 循环迭代

🧠 AI 能力层

RAG Pipeline · 混合检索 · Agent 推理 · 多模型路由

🔌 基础设施层

向量库 · PostgreSQL · Redis · 插件市场

💡 一句话定位：Dify 的出发点是"我要构建一个 AI 应用，需要 RAG、Agent、Prompt 编排"—— 核心是让 LLM 变得可用、可管理、可上生产。

核心特性一览

🎯 模型中立GPT-4 / Claude / DeepSeek / 通义千问 一键切换，业务逻辑零改动
📚 完整 RAG 引擎文档导入 → 自动分块 → Embedding → 混合检索 → Reranking
🔧 可视化 Workflow拖拽式 DAG 编排，条件分支 + 循环 + 代码节点 + 人工审批
🤖 Agent 框架ReAct / Function Calling / Plan-and-Execute 多种推理模式
🔌 插件市场100+ 模型 + 工具插件，官方仓库 + 社区贡献
📊 LLMOps 观测日志追踪 · 成本分析 · 标注改进 · A/B 测试 Prompt
🌐 内置 WebApp零前端即可发布对话界面，支持嵌入 iframe / SDK
📡 RESTful API完整 API，可被其他系统调用 —— 与 n8n 协同的关键

RAG 知识库引擎 —— Dify 的王牌

这是 Dify 最出彩的能力。一套完整的 RAG Pipeline，从文档上传到检索生成的每一步都可通过可视化界面配置 —— 不需要写 Python、不需要调 LangChain。

📄 文档上传→ ✂️ 智能分块→ 🧮 Embedding→ 🗄️ 向量存储→ 🔍 混合检索→ 📊 Reranking→ 💬 LLM 生成

RAG 配置要点

📥 文档格式PDF / Word / PPT / Markdown / TXT / 网页 URL / Notion
✂️ 分块策略自动分块 / 自定义长度 · 父子分块 · Q&A 模式
🧮 Embedding 模型OpenAI text-embedding-3 / BGE-M3 / Jina / Cohere
🗄️ 向量数据库Weaviate / Milvus / Qdrant / pgvector / Chroma
🔍 检索模式向量检索 · 关键词 BM25 · 混合检索 · Top-K 可配
📊 RerankingCohere Rerank / Jina Reranker 二次排序提升精度
dify 知识库 API # 通过 API 向知识库上传文档 curl -X POST "https://your-dify/v1/datasets/{dataset_id}/documents" \ -H "Authorization: Bearer {api_key}" \ -F "file=@financial_model_guide.pdf" \ -F "indexing_technique=high_quality" # 检索知识库 curl -X POST "https://your-dify/v1/datasets/{dataset_id}/retrieve" \ -H "Authorization: Bearer {api_key}" \ -d '{"query":"WACC 如何计算？","retrieval_model":{"search_method":"hybrid_search"}}'

工作流节点类型全解

Dify Workflow 基于 DAG（有向无环图）结构，通过拖拽节点、连线来编排 AI 逻辑。以下是所有核心节点：

分类节点功能说明
基础 Start 定义输入变量（文本 / 段落 / 数字 / 文件），触发工作流入口
End / Answer 直接回复结果 或 声明变量输出给下游 API 调用者
AI LLM 调用大模型，支持 System/User Prompt + 变量 + 上下文注入 + 记忆窗口
Knowledge Retrieval 从知识库检索相关文档片段，返回 Top-K + Score
Question Classifier 用 LLM 对用户输入分类，输出到不同分支处理
Parameter Extractor 从自然语言中提取结构化参数（如"计算茅台 2025 DCF" → {股票:茅台, 年份:2025, 方法:DCF}）
逻辑 IF/ELSE 条件分支，支持 等于 / 包含 / 为空 / 正则 等操作符
Code 执行 Python / JavaScript 代码，做复杂数据转换
Template Transform 用 Jinja2 模板语法拼接字符串
Variable Aggregator 聚合多路输入变量为一个输出
Iteration 对数组逐条执行子流程（如批量生成 10 家公司的估值摘要）
工具 HTTP Request 调用任意 REST API（获取行情数据、查询数据库等）
Tool (插件) 调用已安装的插件工具（搜索 / 计算 / 画图等）
Google / Tavily Search 联网搜索获取实时信息
Assigner 为会话变量赋值（如设置 user_role="analyst"）

实战案例：DCF 估值智能助手

需求：内部估值团队需要一个 AI 助手 —— 能读懂公司财报、解释 DCF/WACC 概念、用搜索工具获取最新市场数据、计算估值结果。

Step 1：创建 Chatflow

Dify 首页 → 创建应用 → 选择 Chatflow → 命名「DCF 估值助手」

Step 2：Workflow 设计

🏁 Start
query: 用户问题→ 🔀 问题分类
LLM 判断意图↙↘ 📚 知识库
检索概念 🌐 联网搜索
获取数据 ↓↓ 🧠 LLM 综合
整合回答→ 💬 Answer
输出结果

Step 3：关键节点配置

问题分类节点 System Prompt 你是一个智能路由助手。根据用户输入判断类型： - "概念解释"：用户在问估值模型理论（DCF/WACC/CAPM 含义） - "数据查询"：用户在问具体数据（某公司市值/财报/股价） - "估值计算"：用户要求计算某公司的估值 只回复以上三个标签之一，不要任何额外文字。
代码节点：DCF 计算 (Python) def main(fcff: float, wacc: float, growth: float, terminal_g: float = 0.03) -> dict: """计算 DCF 企业价值""" projections = [] total_pv = 0 fcf = fcff # 预测期（5 年） for year in range(1, 6): fcf *= (1 + growth) pv = fcf / ((1 + wacc) ** year) projections.append({"year": year, "fcf": round(fcf,2), "pv": round(pv,2)}) total_pv += pv # 终值（永续增长模型） terminal_value = (fcf * (1 + terminal_g)) / (wacc - terminal_g) terminal_pv = terminal_value / ((1 + wacc) ** 5) return { "projections": projections, "terminal_value": round(terminal_value, 2), "terminal_pv": round(terminal_pv, 2), "enterprise_value": round(total_pv + terminal_pv, 2) }
LLM 综合节点 System Prompt 你是一位资深金融估值分析师。请根据以下信息回答用户问题： 【知识库检索结果】 {{knowledge_context}} 【搜索结果】 {{search_results}} 【估值计算结果】 {{dcf_result}} 【要求】 1. 先给出简洁的结论 2. 展开详细分析（引用数据来源） 3. 如果有计算，展示关键步骤 4. 用 Markdown 格式，表格展示数据

最终效果

✅ 用户输入 "帮我算一下茅台 2025 年的 DCF 估值，假设 FCFF 800 亿，WACC 9%" → Agent 自动分类 → 联网搜最新数据 → 用 Code 节点计算 → LLM 生成含表格的完整估值报告。整个过程 0 行前端代码。

n8n — 加了 AI 能力的自动化引擎

n8n（取自 "Nodes to Nodes"）是一个公平代码（Fair-code）的工作流自动化平台，构建在 Node.js 之上。核心哲学：一切皆节点 —— HTTP 请求、数据库查询、AI 调用、代码执行，全用可视化画布串联。

2025 年获得 112k GitHub Stars（JavaScript Rising Stars 总冠军），1.8 亿美元 C 轮融资，估值 25 亿美元。近年持续加强 AI 能力：AI Agent Builder、LangChain 原生集成、AI Workflow Builder（自然语言生成工作流）。

💡 一句话定位：n8n 的出发点是"我要自动化一个业务流程，其中某些步骤可以用 AI"—— 核心是连接万物，AI 是能力之一而非全部。

核心特性一览

🔗 1200+ 内置节点Slack · Notion · PostgreSQL · GitHub · AWS · OpenAI · 几乎覆盖所有 SaaS
🎨 可视化画布拖拽连接节点，实时预览每步输出，数据流透明可追踪
🤖 AI Agent Builder创建带记忆、工具调用、guardrails 的 AI Agent
🧪 LangChain 集成原生支持 LangChain chain，混合无代码 + 有代码 AI 链
🖥️ 自托管 = 完全免费无功能限制、无执行次数限制（企业版按需）
⚡ 单实例 220+/秒企业级并发，适合高吞吐数据管道
🔄 版本控制工作流版本化发布 + 回滚 + 多人协作冲突检测
📡 Webhook / API原生 HTTP 触发器 + REST API，可被任意系统调用

节点体系全解

1. 触发器节点（决定"什么时候跑"）

节点说明金融场景

Merge合并并行分支的数据，支持多种合并策略（append / merge by key / wait）
Loop / While循环执行直到条件不满足
Execute Workflow调用子工作流，实现模块化架构

3. 数据处理节点

节点说明

HTTP Request调用任意 REST API（GET/POST/PUT/DELETE），支持 OAuth2 / Bearer Token / API Key
Set / Function设置变量、数据映射、字段重命名
Date & Time日期格式化、时区转换、时间计算

4. 代码节点实战

n8n Code 节点 · JavaScript // 从多个数据源拼装估值参数 const items = $input.all(); for (const item of items) { // 从上一个节点拿到股票数据 const { ticker, marketCap, totalDebt, cash, ebitda } = item.json; // 计算 EV/EBITDA 倍数 const enterpriseValue = marketCap + totalDebt - cash; const evEbitdaMultiple = (enterpriseValue / ebitda).toFixed(2); // 估值区间判断 let signal = '中性'; if (evEbitdaMultiple < 10) signal = '低估 🟢'; else if (evEbitdaMultiple > 20) signal = '高估 🔴'; item.json.evEbitda = evEbitdaMultiple; item.json.signal = signal; } return items;

需求：每个交易日收盘后，自动拉取自选股数据 → AI 分析估值变化 → 生成 HTML 报告 → 发送到飞书群。

工作流全景

⏰ 18:00 定时→ 📡 HTTP 拉行情
腾讯自选股 API→ 🔄 分批处理
Split in Batches→ 🧠 AI 分析
LLM 节点→ 📊 汇总拼接
Code 节点→ 🔀 判断信号
IF 节点→ 📨 飞书通知
HTTP POST

关键节点配置

Code 节点：估值报告 HTML 生成 // 接收 AI 分析结果，生成 HTML 报告 const analyses = $input.all(); const date = new Date().toLocaleDateString('zh-CN'); let rows = analyses.map(a => { const d = a.json; const change = d.changePct > 0 ? `🔴+${d.changePct}%` : `🟢${d.changePct}%`; return `<tr> <td>${d.ticker}</td> <td>${d.name}</td> <td>¥${d.price}</td> <td>${change}</td> <td>${d.aiSummary}</td> </tr>`; }).join(''); const html = `<h2>📊 每日估值报告 — ${date}</h2> <table border="1" style="border-collapse:collapse"> <tr><th>代码</th><th>名称</th><th>现价</th><th>涨跌幅</th><th>AI 分析</th></tr> ${rows} </table> <p>⚠️ 本报告由 AI 自动生成，仅供参考，不构成投资建议。</p>`; return [{ json: { reportHtml: html, date } }];
LLM 节点 System Prompt 你是一个资深股票分析师。请根据当前行情数据分析估值变化： 股票名称：{{name}} 代码：{{ticker}} 现价：{{price}} 涨跌幅：{{changePct}}% PE(TTM)：{{pe}} PB：{{pb}} 市值：{{marketCap}} 亿 请输出 JSON 格式： { "ticker": "代码", "name": "名称", "price": 现价, "changePct": 涨跌幅, "aiSummary": "一句话分析（含估值判断和关键因素，不超过50字）" } 只输出 JSON，不要任何额外内容。
飞书 Webhook 通知 # n8n HTTP Request 节点配置 Method: POST URL: https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_WEBHOOK Body: { "msg_type": "interactive", "card": { "header": { "title": { "content": "📊 每日估值报告", "tag": "plain_text" } }, "elements": [ { "tag": "markdown", "content": "{{reportHtml}}" } ] } }
✅ 这个工作流部署后完全自动运行：每个交易日 18:00 → 自动拉数据 → AI 分析 → 生成报告 → 推送飞书卡。整个过程不需要任何人工干预。

部署与最佳实践

Docker 快速部署

bash # 快速体验（数据不持久） docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n # 生产部署（数据持久化） docker run -d --name n8n -p 5678:5678 \ -v n8n_data:/home/node/.n8n \ -e N8N_BASIC_AUTH_ACTIVE=true \ -e N8N_BASIC_AUTH_USER=admin \ -e N8N_BASIC_AUTH_PASSWORD=你的强密码 \ -e N8N_ENCRYPTION_KEY=你的加密密钥 \ -e N8N_HOST=your-domain.com \ -e N8N_PROTOCOL=https \ -e GENERIC_TIMEZONE=Asia/Shanghai \ n8nio/n8n

六条核心最佳实践

🧩 模块化设计避免 50 个节点的巨型工作流 → 拆分为「主流程 + Execute Workflow 子模块」
🛡️ 错误处理关键节点启用 Continue On Fail + Error Trigger → Slack 告警
📡 Webhook 优先避免轮询！Webhook 实现秒级响应，显著降低系统负载
📦 批量处理使用 Split in Batches 分批（每批 100 条），避免内存溢出
🔐 凭证安全永不硬编码密钥！使用凭证系统或 $env.VAR_NAME 环境变量
📝 版本控制导出 JSON 存入 Git，分环境管理（dev → staging → prod）
